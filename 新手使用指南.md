## 🧠 一、 Time-LLM 底层逻辑：它是如何实现预测的？

对于新手来说，最大的疑问往往是：“语言模型明明是用来处理文本（词语）的，它怎么能看懂全是数字的时间序列，甚至预测未来的数值？”

Time-LLM 的核心思想叫作 **“模型重编程”（Model Reprogramming）**。它并没有修改 LLM 内部的权重，而是通过一个巧妙的“翻译器”，把时间序列数据伪装成 LLM 能看懂的“外语”。

具体流程可以拆解为以下几步：

1. **分块与切片（Patching）**
   - 我们的时间序列数据往往很长，如果每个时间点的数值都直接输入，不仅冗余而且计算量巨大。
   - Time-LLM 首先将长长的时间序列切分成一个个小片段（Patches），就像把一篇文章分成一个个句子。每个片段包含了一定时间窗口内的局部语义信息。
2. **文本原型重编程（Text-prototype Reprogramming）**
   - 这是最核心的一步！切分好的数字片段需要转换成 LLM 熟悉的语言形式。
   - Time-LLM 维护了一个“词表”（可以理解为一些预定义的文本特征向量）。它通过注意力机制（Attention），让时间序列片段去和这个词表中的元素对齐。
   - 结果是，原来冷冰冰的数字片段，被“翻译”成了由文本特征组合而成的向量表示，从而“骗”过了 LLM，让它以为自己处理的还是某种特殊的文本序列。
3. **提示词增强（Prompt-as-Prefix）**
   - 为了让 LLM 更好地理解任务背景，Time-LLM 还会给数据加上“提示词（Prompt）”。
   - 例如，它会告诉 LLM：“这是一个关于交通流量（或者天气、股票）的数据，数据的时间跨度是什么，我们需要预测未来的多少个点”。这些提示词作为前缀，和上一步翻译好的时间序列向量拼接到一起。
4. **大语言模型推理（LLM Processing）**
   - 拼装好的输入被送入冻结了参数（Frozen）的 LLM（比如 Llama、GPT-2 或 BERT）。
   - 由于 LLM 本身具备极强的上下文推理和模式识别能力，它能够从这些“时间序列语言”中捕捉到周期性、趋势性等规律。
5. **投影输出（Output Projection）**
   - LLM 输出的仍然是高维的隐藏状态（Hidden States）。
   - 此时，在模型的尾部，Time-LLM 接了一个简单的线性层（Linear Layer/Projector），它的作用是将 LLM 的输出重新“翻译”回我们需要的预测数值。

**总结一句话**：Time-LLM 把时间序列**打包**并**翻译**成 LLM 懂的话，加上**提示词**让 LLM 帮忙思考，最后再把 LLM 的思考结果**翻译**回数字预测值。这个过程中，LLM 本身是不更新参数的，我们训练的只是两头的“翻译器”。

## 🛠️ 二、 环境配置（保姆级步骤）

跑通深度学习项目，环境是第一关。推荐使用 **Linux 环境** 以及多张显卡（以满足大模型如 Llama 的显存需求），但如果使用小模型（如 GPT-2），单卡也能跑。

### 1. 安装 MiniConda
如果你的服务器还没有安装 Conda，请先安装（以 Linux 为例）：
```bash
wget https://repo.anaconda.com/miniconda/Miniconda3-latest-Linux-x86_64.sh
bash Miniconda3-latest-Linux-x86_64.sh
# 一路回车和 yes，安装完后重启终端或 source ~/.bashrc
```

### 2. 创建并激活虚拟环境
为了不和其他项目冲突，我们创建一个专门给 Time-LLM 用的环境（要求 Python 3.11）：
```bash
conda create -n timellm python=3.11 -y
conda activate timellm
```

### 3. 安装依赖包
项目根目录下有一个 `requirements.txt` 文件，里面列出了所有需要的库。
```bash
# 进入项目根目录
cd Time-LLM

# 安装依赖
pip install -r requirements.txt
```
*提示：安装可能需要几分钟。如果在国内下载慢，可以加上清华源：`-i https://pypi.tuna.tsinghua.edu.cn/simple`*

## 📂 三、 数据集准备

由于官方没有把庞大的数据集直接放在代码仓库里，你需要手动下载并放在正确的位置。

1. **获取数据**：
   点击官方提供的 [Google Drive 链接](https://drive.google.com/file/d/1NF7VEefXCmXuWNbnNe858WvQAkJ_7wuP/view?usp=sharing) 下载数据集压缩包。
2. **放置数据**：
   - 将下载的压缩包解压。
   - 在项目的根目录下（即和 `README.md` 同级的目录）确保有一个名为 `dataset` 的文件夹。
   - 将解压后的所有数据文件夹（如 `ETT-small`, `weather` 等）放入 `dataset` 文件夹中。

正确的数据目录结构看起来应该是这样的：
```text

│   ├── ETT-small/

│   │   ├── ETTh2.csv
│   │   └── ...

│   │   └── weather.csv
│   └── ...
├── scripts/
├── models/
├── run_main.py
└── ...
```

## 🚀 四、 运行预测模型

项目提供了非常方便的 Bash 脚本（在 `scripts/` 目录下），这些脚本里已经配置好了各种超参数。

### 1. 初次尝试：运行 ETTh1 数据集
以预测电力变压器温度数据（ETTh1）为例，脚本已经为你写好了所有参数。

在终端中，直接运行：
```bash
bash ./scripts/TimeLLM_ETTh1.sh
```

### 2. 脚本里的参数代表什么意思？
如果你打开 `./scripts/TimeLLM_ETTh1.sh` 这个文件，你会看到一堆神秘的命令，这里为你拆解几个最关键的：

- `accelerate launch ...`：因为 LLM 比较大，项目使用了 Huggingface 的 `accelerate` 库来进行多卡分布式训练。`--num_processes 8` 代表使用 8 张显卡。**如果你只有 1 张显卡或者想换数量，一定要修改这里！**（例如改为 `--num_processes 1` 或者干脆去掉 `accelerate` 的相关前缀，直接 `python run_main.py ...`，但推荐使用 `accelerate` 进行显存管理）。
- `--model_name TimeLLM`：使用的模型架构。
- `--root_path ./dataset/ETT-small/` 和 `--data_path ETTh1.csv`：告诉代码去哪里找数据。
- `--seq_len 512`：输入给模型的历史序列长度（比如过去 512 个小时的数据）。
- `--label_len 48`：用于提供上下文帮助预测的已知标签长度。
- `--pred_len 96`：我们要预测的未来序列长度（例如预测未来 96 个小时）。
- `--llm_model LLAMA`：默认使用的底层大语言模型是 Llama。如果你觉得 Llama 太大跑不动，可以在脚本里加上参数 `--llm_model GPT2 --llm_dim 768` 来切换成轻量级的 GPT-2 模型。

### 3. 查看训练进度和结果
代码运行后，终端会打印出类似下面的信息：
```text
    iters: 100, epoch: 1 | loss: 0.3582...
    speed: 1.05s/iter; left time: ...
```
这说明模型正在疯狂地学习和对齐特征。
训练结束后，模型会自动在验证集和测试集上进行评估，并打印出 **Test Loss** 和 **MAE Loss**，这就代表了预测的误差，数值越小，预测越准！预测生成的权重默认保存在 `./checkpoints/` 目录下。

## 💡 常见问题 (FAQ)

**Q1: 报错 `CUDA Out of Memory` (显存不足) 怎么办？**
- **方法一**：调小脚本里的 `batch_size`（比如从 24 降到 8 或 4）。
- **方法二**：更换底座模型。Llama 需要很大的显存。尝试在运行命令中加入 `--llm_model GPT2 --llm_dim 768` 或者 `--llm_model BERT --llm_dim 768`。

**Q2: 报错找不到文件 `No such file or directory: './dataset/ETT-small/ETTh1.csv'`**
- 请仔细检查你第三步【数据集准备】是否把路径放对了，注意大小写和文件夹层级。

**Q3: 代码会自动下载大模型吗？**
- 是的，首次运行代码时，HuggingFace 库会自动从网络上下载 Llama / GPT-2 等模型的预训练权重。请保持服务器网络畅通。如果服务器无法连外网，你需要提前在有网的机器上下载模型权重，并修改代码中的路径指向本地模型。
